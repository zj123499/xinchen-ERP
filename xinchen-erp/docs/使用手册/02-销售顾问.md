# 销售顾问使用手册

> ⚠️ **当前角色状态（2026-07-18）**：系统目前仅启用「超级管理员」一个角色，本角色尚未在「系统设置 → 角色权限」中配置。以下内容为该角色的**预期使用说明**，待角色重新启用后生效；当前阶段相关操作由超级管理员统一处理。

## 角色定位

负责从「线索」到「签约」的前端转化：录入与跟进线索、建立学生档案、推进签约（合同 / 订单）、进行客户回访。

## 可见菜单

工作台、销售管理（线索管理 / 学生档案 / 跟进记录）、合同订单（合同管理 / 订单管理）、客户回访。

## 核心工作流

### 1. 线索管理（线索管理 `/leads`）
- **新增线索**：填写姓名、手机号、微信、来源渠道、意向等级等。系统对手机号 / 微信做唯一性校验，避免撞单。
- **撞单处理**：若手机号已存在，系统提示归属人；如属转介绍，可发起「转介绍申诉」，上传证据后由主管审批划转。
- **跟进**：在线索详情记录跟进动作，或进入「跟进记录」统一维护时间线。
- **线索移交**：可将线索移交给其他顾问，移交需经审批流（LEAD_TRANSFER）通过后变更归属。

### 2. 学生档案（学生档案 `/students`）
- 线索转化 / 直接建档后，维护学生 360° 档案：基础信息、学术成绩（GPA / 雅思 / 托福 / GRE / GMAT）、留学目标（国家 / 专业 / 预算 / 入学季）、生命周期状态。
- 一个学生页聚合：基本信息、跟进时间线、合同、申请项目、Offer、签证、付款、文件、操作日志、风险标记。

### 3. 合同与订单（合同管理 / 订单管理）
- **合同管理**：新建合同，绑定销售顾问 / 咨询顾问 / 文书顾问等角色，填写金额与条款；优惠幅度 >20% 需提交「合同优惠审批」走多级审批。
- **订单管理**：合同关联生成订单，跟踪订单状态直至「结案」。

### 4. 客户回访（客户回访 `/visits`）
- 按系统配置的回访间隔（签约后 / 申请中 / 入学后）创建回访计划与记录，维护客户满意度。

## 按钮权限边界

- **可**：线索 / 学生 新增、编辑、查看；合同 新增、查看；回访 新增、查看。
- **不可**：删除线索 / 学生、审批合同；查看 / 导出财务（敏感财务字段受字段组权限限制可能不可见）。

## 注意事项

- 联系方式属敏感字段，批量导出受「行为熔断」监控（异常导出会触发账号锁定）。
- 离职前名下未签约线索需进入「待离职盘点池」交接，避免客户流失。
- 线索来源、意向等级等请从数据字典选择规范值，保证统计口径一致。
