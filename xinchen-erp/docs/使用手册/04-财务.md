# 财务使用手册

> ⚠️ **当前角色状态（2026-07-18）**：系统目前仅启用「超级管理员」一个角色，本角色尚未在「系统设置 → 角色权限」中配置。以下内容为该角色的**预期使用说明**，待角色重新启用后生效；当前阶段相关操作由超级管理员统一处理。

## 角色定位

负责收款、成本、提成、薪资、报销、利润核算与退费管理，是公司经营数据的「守门人」。

## 可见菜单（待角色启用后可见）

工作台、财务管理（收款管理 / 成本管理 / 提成管理 / 薪资管理 / 报销管理）。

## 核心工作流

### 1. 收款管理（收款管理 `/payments`）
- 登记客户收款（首款 / 尾款 / 其他），支持多币种与汇率换算。
- **权责发生制**：填写财务年度（fiscal_year）与财务月份（fiscal_month），确保跨年返佣 / 收入计入正确期间（如 2026 签约、2027 入学后返佣计入 2027）。

### 2. 成本管理（成本管理 `/costs`）
- 登记营销 / 薪资 / 渠道 / 运营等成本，支持成本分摊规则。

### 3. 提成管理（提成管理 `/commissions`）
- 系统按「分段提成释放引擎」在里程碑（首款 / 尾款 / Offer / 入学 / 返佣）按比例释放提成。
- 提成规则支持**版本控制**：历史订单按签约时生效的规则计算，规则调整不影响历史。

### 4. 薪资管理（薪资管理 `/salaries`）
- 核算并发放员工薪资；退费扣回的负向提成会按已释放比例从下月薪资扣除。

### 5. 报销管理（报销管理 `/reimbursements`）
- 员工提交报销 → 触发 REIMBURSEMENT 审批 → 通过后打款（PAID）。

### 6. 退费与扣回
- 退费申请提交触发 REFUND 审批；审批通过按退费比例生成负向提成（扣回各角色已释放提成）。

### 7. 利润与应收应付（报表 / 查看）
- 利润模型：利润 = 收款 − 成本，可按 国家 × 院校 × 顾问 × 渠道 × 月度 切分。
- 应收（receivables）/ 应付（payables）/ 发票（invoices）/ 其他收入（incomes）模块支持多币种管理。

## 按钮权限边界（预期）

- **可**：收款 新增 / 查看 / 导出；报表 查看。
- **不可**：删除合同、修改提成规则（需管理员或按配置权限）。

## 注意事项

- 多币种请务必填对汇率与币种，否则利润口径失真。
- 退费审批务必与业务确认，扣回逻辑会自动影响相关顾问薪资。
- 敏感财务数据受字段组权限保护，越权查看会被审计记录。
