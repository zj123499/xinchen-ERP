# 部门主管使用手册

## 角色定位

管理本部门的数据与团队：审批下属的线索移交 / 合同优惠 / 报销 / 退费，监管本部门销售与交付进度，处理风险与转介绍申诉。

> **重要说明**：部门主管角色默认**未绑定菜单**，需由超级管理员在「系统设置 → 角色权限」中为其勾选所需菜单（通常含销售 / 交付 / 财务的查看类菜单），并配置「数据权限 = 本部门」。以下为推荐配置与用法。

## 推荐可见菜单（由管理员配置）

工作台、销售管理（线索 / 学生 / 跟进）、合同订单、交付管理（申请 / Offer / 签证）、客户回访、财务管理（查看类）、风险管理、审批中心、客户成功（满意度 / 投诉 / 转介绍）。

## 核心工作

### 1. 审批中心（审批流）
- 处理待办审批：
  - **线索划转（LEAD_TRANSFER）**：移交人提交后，主管审批通过才变更归属。
  - **合同优惠（CONTRACT_DISCOUNT）**：优惠 >20% 多级审批。
  - **报销（REIMBURSEMENT）**、**退费（REFUND）**：按流程逐级审批。
- 支持会签 / 或签；部分高级特性（加签、条件分支、超时自动）待完善。

### 2. 团队监管
- 数据权限限「本部门」，可查看本部门线索 / 学生 / 合同 / 申请 / 收款全貌。
- 通过 BI 看板观察本部门漏斗、人效、风险。

### 3. 风险与转介绍
- **风险看板**：处理本部门高风险学生（材料未完成 / 签证延期 / 付款异常 / 投诉）。
- **转介绍申诉（lead_appeals）**：审批下属的撞单申诉，上传证据后划转或驳回。

### 4. 待离职盘点（如启用）
- 员工离职前，将其未签约线索一键转入交接池，按规则转交并完整继承跟进历史。

## 注意事项

- 主管的数据范围由管理员设定的「数据权限」决定；若看不到预期数据，请联系管理员核对角色配置。
- 审批操作会回写业务状态（如退费审批通过 → 状态变更），请谨慎决策。
- 部门主管菜单需管理员显式开通，新入职主管首次登录若只见「工作台」，属正常，请通知管理员配置。
