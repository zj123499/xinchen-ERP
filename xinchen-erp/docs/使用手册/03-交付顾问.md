# 交付顾问使用手册

## 角色定位

负责签约后的「交付」环节：申请递交、文书、Offer 跟进、签证办理，直到学生到校注册。

## 可见菜单

工作台、交付管理（申请管理 / Offer 管理 / 签证管理）、客户回访。

## 核心工作流

### 1. 申请管理（申请管理 `/applications`）
- 为学生创建申请项目：目标院校、专业、学位、入学季。
- 维护申请状态流转：**材料准备 → 递交申请 →（Offer）→ 递交签证 → 到校注册 → 结案**。
- **材料管理**：跟踪申请材料完成度（application_materials）。
- **文书任务**：通过「AI 文书助手」生成 PS / CV / 推荐信初稿，并回写至文书任务（copywriter_tasks）。

### 2. Offer 管理（Offer 管理 `/offers`）
- 登记收到的 Offer，关联对应申请；Offer 到达是提成释放的里程碑之一。

### 3. 签证管理（签证管理 `/visas`）
- 登记签证递交与状态（递交 / 批准 / 驳回 / 申诉）；签证批准对应「到校注册」节点。

### 4. 客户回访
- 与销售协同进行申请 / 签证阶段回访。

## 进度可视化

- **数据大屏**按 5 节点展示全部学生申请进度与滚动墙，交付顾问可据此优先处理临近截止或卡点学生。
- **风险管理中心**会对「材料未完成且临近截止」「签证长期无结果」等自动标记风险。

## 按钮权限边界

- **可**：申请 新增、编辑、查看；回访 查看。
- **不可**：查看财务 / 提成等（受角色限制）。

## 注意事项

- 申请状态需如实更新，数据大屏与 BI 看板依赖这些状态驱动经营分析。
- 接近申请截止日期前务必确认材料完成率，规避高风险标记。
- 可善用「AI 选校顾问 / 文书助手」提升方案专业度与效率（产出可固化到业务流）。
