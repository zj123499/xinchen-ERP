系统定位

新辰ERP是为新辰留学量身定制的一站式企业资源管理系统，覆盖留学服务全业务流程：从市场获客→销售转化→合同签约→交付执行→客户回访→财务核算→风险管控，实现数据驱动的精细化运营。

🎯 核心目标

  • 业务流程全覆盖：线索→合同→交付→财务
  • 角色权限精细化：15个角色按模块隔离数据
  • 经营数据可视化：实时驾驶舱+数据大屏
  • 自动化工作流：审批、提醒、定时任务

👥 用户群体

  • 市场团队：线索采集、新媒体运营
  • 销售团队：客户跟进、合同签约
  • 交付团队：申请递交、签证办理
  • 财务团队：收款核算、报表分析
  • 管理层：经营驾驶舱、全局把控

🚀 三、部署指南

部署命令

# 1. 打包本地源码
cd xinchen-erp
tar -cf /tmp/xinchen_src.tar --exclude=node_modules --exclude=.next \
  --exclude=.git --exclude=.env --exclude=.env.local .

# 2. 上传到服务器
scp /tmp/xinchen_src.tar ubuntu@111.229.72.128:/home/ubuntu/

# 3. 解压并构建
ssh ubuntu@111.229.72.128
cd /home/ubuntu/xinchen-erp
tar -xf ../xinchen_src.tar
docker compose build
docker compose up -d

# 4. 查看日志
docker compose logs -f --tail=50
注意：首次部署需确保 ansible 已在服务器上完成用户创建。日常更新使用 docker compose build（保留缓存），prisma db push 会在容器启动时自动执行。

Docker Compose 配置

services:
  xinchen-erp:
    build: .
    container_name: xinchen-erp
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://user:pass@1Panel-postgresql-Bq8Z:5432/erp
      - JWT_SECRET=your-secret
    networks:
      - 1panel-network
    restart: always

通用规范

  • 认证：除 /api/auth/login 外全部接口需携带 Authorization: Bearer <token> 或 Cookie token
  • 租户隔离：所有数据根据 JWT 中的 tenantId 自动过滤
  • 分页：列表接口支持 ?page=1&pageSize=20 查询参数
  • 权限：操作类接口（create/update/delete）校验角色权限编码，无权限返回 403
  • 响应格式：{ list: [], total: N, page: P } 或 { error: "message" }

🛠️ 八、运维常见问题

Q1: 部署后页面 404？

检查 Nginx 是否指向正确端口（3000），Docker 容器是否运行：docker ps | grep xinchen

Q2: 数据库连接失败？

检查容器是否在同一网络：docker network inspect 1panel-network。确认 DATABASE_URL 中的主机名与数据库容器名一致。

Q3: 磁盘空间满？

# 定期清理 Docker 缓存
docker builder prune -af
docker image prune -af
df -h /

Q4: 构建失败（Prisma 错误）？

检查 Schema 文件是否有双向关系缺失。Prisma 7.8 要求所有 @relation 必须有反向字段。

Q5: 新员工如何创建登录账号？

在「员工管理」新增员工时填写手机号，系统自动创建账号（用户名=手机号，默认密码 Xc@123456，首次登录强制改密码）。

Q6: 员工离职怎么处理数据？

将员工状态改为「离职」，系统自动停用登录权限。管理员在员工列表点击「重新分配数据」将离职员工名下的线索/学生/订单转给在职员工。

Q7: 看板数据看不到？

确认角色已配置 view 权限。管理员在「员工管理」点击「配置权限」一键补全。

Q8: 构建卡住 / npm ci 超时？

去掉 --no-cache 参数，使用缓存构建。仅更新源码文件时 Docker 层缓存可复用。

Q9: Turbopack 构建失败（Alpine）？

Next.js 16 默认 Turbopack 在 Alpine (musl) 上无原生绑定。需确保 package.json 中 "build": "next build --webpack"。

Q10: 查看容器日志？

docker compose logs -f --tail=100 xinchen-erp
